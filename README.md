# 2026 NFL Draft Dashboard

A head-to-head (you vs. Claude) draft command center. Dark NFL broadcast aesthetic, persistent via `localStorage`, ship-able to Vercel as a static site.

## Stack

- Vite + React + TypeScript
- Tailwind CSS
- `@dnd-kit/core` + `@dnd-kit/sortable` for drag-to-rank
- `lucide-react` for icons
- `node-fetch` + `cheerio` for the RAS scraper

## Setup

```bash
npm install
npm run dev        # http://localhost:5173
```

## Sections

1. **Big Boards** — toggle between **My Board** (drag-sort top-150 with editable slots, top 50 = "the board" and 51–150 = "the pool") and **Claude's Top 50** (composite score: RAS × production × draft-capital proxy).
2. **Predicted Mock** — what each of us thinks teams *will* do. `MINE` is pre-seeded from a PFSN sim screenshot and editable; `CLAUDE'S` is static JSON.
3. **GM Mock** — what each of us *would* do if we were each team's GM. Includes per-pick reasoning on Claude's side.

Each mock tab has a `MINE ↔ CLAUDE'S` segmented toggle. The "Mine" side is editable in-app with localStorage persistence; the "Claude's" side loads from JSON and is edited by hand.

## Editable data files

All live in `/data`. Edit by hand to update content — the Vite dev server will hot-reload.

| file | what it is |
| --- | --- |
| `players2026.json` | 150 prospect slots for the user's big board — first 20 seeded, rest blank |
| `myPredictedMock.json` | User's predicted first round (seeded from PFSN) |
| `myGMMock.json` | User's GM-mode picks |
| `claudesPredictedMock.json` | Claude's prediction of what teams will do |
| `claudesGMMock.json` | Claude's "what I would do" picks + 2–3 sentence reasoning |
| `productionStats.json` | PFF grades / stat blurbs keyed by player name (drives Section 5 scoring) |
| `rasScores.json` | Generated by `npm run fetch-ras`; starts as `{}` |

`localStorage` only holds your in-UI edits (board ordering, mock picks). JSON files are the source of truth for initial state.

## RAS scraper

`ras.football` has no public API. The scraper in `scripts/fetchRAS.ts`:

- Searches for each player's page via the WordPress search endpoint
- Parses RAS, 40, vertical, height/weight, etc. via `cheerio`
- Caches raw HTML under `data/.ras-cache/` (gitignored) so reruns don't re-hit the site
- Throttles requests with a 1.5s delay

**Please be respectful** — don't rerun unnecessarily. Many 2026 prospects haven't combined yet; those entries store `ras: null` and the UI shows `RAS: pending combine`.

```bash
npm run fetch-ras
```

## Scoring model (Section 5)

`src/lib/prospectScore.ts` blends three inputs per prospect:

1. **RAS** (0–10 → scaled 0–100)
2. **Production proxy** (PFF grade or tier from `productionStats.json`)
3. **Draft capital** (average ADP across all four mocks in this app)

Weights shift by position (see the Methodology panel on the Top 50 tab). RAS correlates strongly with NFL success at EDGE / CB / S, weakly at QB / RB. Players missing RAS data have their RAS weight redistributed into the other two components.

## Deploying to Vercel

This app is a static SPA — Vercel auto-detects Vite.

**Via the Vercel dashboard (recommended):**

1. Push this branch to GitHub (already done if you're reading this).
2. Go to [vercel.com/new](https://vercel.com/new) → import the repo.
3. Framework preset: **Vite** (auto-detected). Build command `npm run build`, output directory `dist`.
4. Leave env vars empty — there are none.
5. Click **Deploy**.

**Via CLI:**

```bash
npm i -g vercel
vercel           # first run: link to project
vercel --prod    # deploy production
```

No serverless functions, no secrets, no backend — just static files under `dist/`. The RAS scraper runs locally only; its output is committed as part of `rasScores.json`.
